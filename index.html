1. Introducción a la simulación

1.1. Definición e importancia de la simulación en la Ingeniería

La simulación es una técnica que permite imitar el funcionamiento de un proceso o sistema real mediante un modelo computacional. En ingeniería, la simulación es crucial porque:

  • Permite analizar sistemas complejos sin alterar el entorno real
  • Reduce costos al evitar implementaciones físicas prematuras
  • Facilita la identificación de cuellos de botella y puntos de mejora
  • Permite evaluar múltiples escenarios en poco tiempo
Ejemplo Práctico:

Simulación del flujo de tráfico en una intersección para determinar la configuración óptima de semáforos que minimice los tiempos de espera.

1.2. Conceptos básicos de la simulación

Para comprender la simulación, es fundamental dominar estos conceptos:

Concepto Definición
Modelo Representación simplificada de un sistema real
Entidad Elemento que se mueve a través del sistema
Atributo Característica de una entidad
Evento Suceso que cambia el estado del sistema
Recurso Elemento que presta servicio a las entidades
1.3. Metodología de la simulación

Pasos para desarrollar un estudio de simulación:

  1. Definición del problema
  2. Diseño del modelo conceptual
  3. Recolección y análisis de datos
  4. Construcción del modelo
  5. Verificación y validación
  6. Experimentación
  7. Análisis de resultados
  8. Implementación
1.7. Ventajas y desventajas de la simulación

Ventajas:

  • Análisis de sistemas complejos
  • Prueba de escenarios hipotéticos
  • Comprensión del sistema
  • Reducción de riesgos

Desventajas:

  • Requiere tiempo y recursos
  • Calidad depende de los datos de entrada
  • Los resultados no son exactos
  • Requiere experiencia para interpretación

2. Números pseudoaleatorios

2.1. Métodos de generación de números Pseudoaleatorios

Los números pseudoaleatorios son secuencias deterministas que aparentan ser aleatorias. Métodos comunes:

Método de Congruencia Lineal

Generador más común: Xn+1 = (aXn + c) mod m

Donde: a = multiplicador, c = incremento, m = módulo

Método de Fibonacci

Xn = (Xn-1 + Xn-2) mod m

Requiere dos semillas iniciales

Ejemplo en Python:
# Generador de números pseudoaleatorios
def linear_congruential(seed, a=1664525, c=1013904223, m=2**32, n=10):
    results = []
    x = seed
    for _ in range(n):
        x = (a * x + c) % m
        results.append(x / m)  # Normalizar a [0,1)
    return results

print(linear_congruential(seed=12345))

5. Proyecto Integrador

5.1. Análisis, modelado y simulación de un sistema

El proyecto integrador consiste en analizar, modelar y simular un sistema o subsistema de servicios o productivo de una empresa para detectar posibles mejoras.

Etapas del proyecto:
  1. Definición del sistema: Selección del sistema a simular (línea de producción, sistema de atención al cliente, cadena de suministro)
  2. Recolección de datos: Tiempos de proceso, tasas de llegada, recursos disponibles
  3. Modelado conceptual: Diagrama de flujo del proceso, definición de entidades, recursos y eventos
  4. Implementación: Desarrollo del modelo en un software de simulación
  5. Experimentación: Pruebas con diferentes configuraciones
  6. Análisis de resultados: Identificación de cuellos de botella y propuestas de mejora
Ejemplo: Simulación de una línea de ensamblaje

Objetivo: Optimizar el flujo de producción reduciendo tiempos de espera y maximizando la utilización de recursos.

Resultados: Reducción del 25% en tiempos de espera mediante redistribución de estaciones de trabajo.

Caso Práctico

Simulación de un sistema de inventarios
Problema:

Una empresa minorista necesita determinar la política óptima de reabastecimiento para minimizar costos manteniendo un nivel de servicio adecuado.

Parámetros del sistema:
  • Demanda diaria: Distribución normal (μ=50, σ=10)
  • Tiempo de entrega: 2 días
  • Costo de ordenar: $50 por pedido
  • Costo de mantenimiento: $2 por unidad por día
  • Costo por faltante: $10 por unidad
  • Inventario inicial: 150 unidades
Modelado y simulación:

Se implementó un modelo de simulación de eventos discretos para evaluar diferentes políticas (s, S):

# Pseudocódigo para simulación de inventarios
Initialize:
    inventory = 150
    time = 0
    order_cost = 0
    holding_cost = 0
    shortage_cost = 0

Events:
    Customer arrival:
        demand = normal(50, 10)
        if inventory >= demand:
            inventory -= demand
        else:
            shortage = demand - inventory
            shortage_cost += shortage * 10
            inventory = 0

    Order arrival:
        inventory += order_quantity

    Daily review:
        if inventory <= reorder_point:
            place_order(order_up_to_level - inventory)
            order_cost += 50
        holding_cost += inventory * 2
Resultados:

La política óptima encontrada fue (s=100, S=300) con un costo diario promedio de $142.
